## 1. Why

Cross-repo comparison against the sibling reference app surfaced three low-cost UX wins the main app
lacked; the user picked the **"✨ changed" indicator** as this round. Today the refine success path
shows `优化中…` then dismisses and the refined text is **never shown back** — the user gets no signal
that a correction happened `[file:Sources/VoiceInput/Core/AppCoordinator.swift]`. (Audio cues and the
error/status-feedback capsule had already shipped in earlier rounds.)

## 2. What shipped

**Feature: `✨ <refined text>` confirmation (changed-only, insert-first).**
- **Scope (③):** shown only when refinement **succeeded AND changed the text** —
  `changed = !outcome.fellBack && trimmed(refined) != trimmed(raw)`
  `[file:AppCoordinator.swift:181-183]`. Not shown for LLM-off, refined-but-identical, empty, or the
  LLM-fallback path. Mutually exclusive with the `ℹ︎` fallback by construction (`refineChanged`
  requires `!fellBack`).
- **Beat (decided with user): insert first, then flash ✨.** The refined text is pasted
  **immediately** (zero added latency — the Fn-release→paste latency is the flagged next-round pain
  point), then the already-visible capsule swaps to `✨ <text>` and auto-dismisses after
  `refinedNoticeDuration = 1.5 s` `[file:AppCoordinator.swift:45,232-244]`. Reuses the existing
  inject-while-visible structure the `ℹ︎` fallback path already shipped with.
- **Glyph:** `✨` (U+2728) in natural **color** presentation — intentionally not the monochrome
  U+FE0E convention of `⚠︎`/`ℹ︎` — via a new `StatusKind.refined`
  `[file:Sources/VoiceInput/UI/FloatingCapsuleWindow.swift:StatusKind]`.
- **No toggle, no new localized string** (✨ is a glyph + the user's own on-screen text); `L10n`
  untouched. Always-on, consistent with the existing error/fallback notices.
- The "changed" comparison lives at the call site (the `Task` closure has both `raw` and `outcome`);
  `RefineOutcome.refined` carries only the refined string, so `LLMRefiner` stays presentation-agnostic.

**Files:** `FloatingCapsuleWindow.swift` (+`StatusKind.refined`), `AppCoordinator.swift` (changed
signal + `finish(…, refineChanged:)` branch).
